String Intern

  • What the different when the creating a string by using:
    String str1 = new String("abc");
    String str2 = new String("abc");
    String str3 = "abc"; // Compile-Time String Interning
    String str4 = "abc"; // Same
    String str5 = new String("abc").intern(); // Run-Time String Interning 
                  -------------------  
             |                   | (Heap)
str1 ------------->  "abc"       | 
             |                   |
str2 ------------->  "abc"       |
             |                   |
             |-------------------|
             |   (String Pool)   |
             |                   |   
str3, str4, -------> "abc"       |
str5         |                   | 
              -------------------
  • Explain:
    String pool is inside Heap. Intern will move string object from heap to String Pool.
    All compile-time constant strings in Java are automatically interned.
  • str1 == str2; // False
    str1 == str3; // False
    str3 == str4; // True
    str3 == str5; // True
    str1.equals(str2); // True
    str1.equals(str3); // True
  • Because str1, str2 and str3 reference different objects, so "==" operator return false.
    Method "equals" will check the whether the fields "value" (char[]) in string equal.
  • Because the literal "abc" is already of type String, using constructor will create an extra unnecessary object. Therefore, double quotes should be used if you just need to create a String. If you do need to create a new object in the heap, constructor should be used.
